Description

The users of the service will be managers and candidates. A manager can create vacancies and read the candidate's resume; a candidate can create a resume and view vacancies. Both of them need a way to log in, log out, and see their profile.

So many pages to make, but first things first: the menu.

Objectives

The menu will be the main page of the service. This page will have the links to all other pages that the user needs. From the menu, you can go to the vacancies list, resumes list, and personal profile. In the head of the page, there will be a greeting message.

Let's look at a simple design.

  1. The menu should contain the <h2> element at the top: 
    <h2>Welcome to HyperJob!</h2>
  2. Five links should go next in separate <div> elements:
    • To login page with the href attribute equals "/login"
    • To sign up page with the href attribute equals "/signup"
    • To vacancies list with the href attribute equals "/vacancies"
    • To resumes list with the href attribute equals "/resumes"
    • To personal profile with the href attribute equals "/home"

If you start the application on your computer with python manage.py runserver command, the menu should be available on the address localhost:8000.

If you want to use a custom directory for your templates, add the path to it to the TEMPLATE['DIRS'] list in the settings.py module.

To combine a template with the HTTP handler, you can use the django.shortcuts.render function.

render(request, template_name)

You don't need to implement the handlers for these pages: you'll do it at the next steps.
